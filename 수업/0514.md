
# 구현 내용

## 1. MySQL 연동하기

- MySQL JDBC Driver 연결
- Java와 DB 연결 설정

---

## 2. DB 연결 클래스 작성

```text
db.DBUtil
```

역할:
- DB 연결 객체 반환
- DriverManager 사용
- Connection 관리

---

## 3. DTO 클래스 만들기

### dto.CustomerDTO
- 고객 정보 저장

### dto.BookDTO
- 도서 정보 저장

### dto.RentalDTO
- 대여 정보 저장

역할:
- DB ↔ Java 데이터 이동 객체

---

## 4. 추가 기능 구현

### 회원 등록 기능

```java
insertCustomer(CustomerDTO dto)
```

기능:
- 회원 이름 저장
- 전화번호 저장

---

### 도서 등록 기능

```java
insertBook(BookDTO dto)
```

기능:
- 도서 제목 저장
- 저자 저장
- 재고 저장

---

## 5. 전체 도서 조회 기능

```java
findAllBooks()
```

출력 예시

```text
번호 : 1
제목 : 자바의 정석
저자 : 남궁성
재고 : 5
```

# 현행 시스템 분석

## 실습 시나리오

한 학원에서 기존 수강 관리 시스템의 문제점을 해결하기 위해 새로운 시스템 구축 프로젝트를 진행하려고 한다.

현재 시스템 문제점

- 모바일 사용이 불편하다.
- 출결 처리 속도가 느리다.
- 문자 발송 실패가 자주 발생한다.
- 상담 이력이 통합 관리되지 않는다.
- 관리자마다 사용하는 프로그램이 다르다.
- 수강료 결제 내역이 자주 누락된다.

---

# 문제 1. 현행 시스템 구성 분석

| 시스템명 | 역할 |
|---|---|
| 수강생 관리 시스템 | 수강생 등록 및 조회 |
| 출결 관리 시스템 | 출석 및 결석 처리 |
| 결제 관리 시스템 | 수강료 결제 |
| 문자 발송 시스템 | SMS 발송 |
| 상담 관리 시스템 | 상담 이력 저장 |

| 상담 관리 시스템 | 상담 이력 저장 | 상담 직원 |

# 외부 시스템 연동 분석

| 외부 시스템 | 송신 데이터 | 수신 데이터 | 연동 목적 |
|---|---|---|---|
| SMS API | 전화번호, 문자내용 | 발송 결과 | 수강생에게 문자 발송 |
| 결제 API | 결제금액, 사용자 정보 | 결제 승인 결과 | 수강료 결제 처리 |
| 이메일 API | 이메일 주소, 메일 내용 | 발송 결과 | 이메일 안내 및 공지 |

# 프론트엔드의 역할

- 사용자 화면 제공
- 입력 데이터 전달
- 결과 화면 출력
- 버튼/메뉴 UI 처리

예시:
- 로그인 화면
- 출결 조회 화면
- 수강생 등록 화면

---

# 백엔드의 역할

- 비즈니스 로직 처리
- 데이터 검증
- API 연동 처리
- 데이터베이스 연결

예시:
- 로그인 처리
- 출결 저장
- 결제 처리
- 문자 발송 요청

---

# 데이터베이스 역할

- 데이터 저장
- 데이터 조회
- 데이터 수정/삭제
- 시스템 정보 관리

예시:
- 수강생 정보 저장
- 출결 기록 저장
- 결제 내역 저장
- 상담 이력 저장

# 시스템 개선 목표

- 기존 핵심 기능 유지
- 문제 기능 개선
- 사용자 편의 기능 추가

---

# 1. 유지해야 하는 기능

## 회원가입
- 현재 정상적으로 동작한다.
- 사용자가 서비스를 이용하기 위해 반드시 필요한 기능이다.

## 장바구니 담기
- 현재 정상적으로 동작한다.
- 상품 구매 과정에서 자주 사용하는 핵심 기능이다.

---

# 2. 개선해야 하는 기능

## 상품 조회 기능

- 현재 문제:
  검색 속도가 느림

- 개선 방향:
  데이터베이스 검색 최적화 및 INDEX를 적용하여 조회 속도를 개선한다.

---

## 로그인 기능

- 현재 문제:
  보안이 부족함

- 개선 방향:
  비밀번호 암호화 및 JWT/세션 인증을 적용하여 보안을 강화한다.

---

## 결제하기 기능

- 현재 문제:
  결제 처리 속도가 느림

- 개선 방향:
  결제 API 처리 로직을 최적화하고 서버 성능을 개선하여 처리 속도를 향상시킨다.

---

## 배송 조회 기능

- 현재 문제:
  모바일 화면이 깨짐

- 개선 방향:
  반응형 UI를 적용하여 모바일 환경에서도 정상적으로 보이도록 수정한다.

---

# 3. 새롭게 추가해야 하는 기능

## 상품 리뷰 기능

- 사용자가 상품 구매 후 리뷰와 평점을 작성할 수 있도록 추가한다.
- 상품 선택에 도움을 주어 사용자 편의성을 높일 수 있다.

---

## 찜하기(관심상품) 기능

- 사용자가 관심 있는 상품을 저장할 수 있도록 추가한다.
- 나중에 다시 쉽게 확인할 수 있어 구매 편의성을 높일 수 있다.
