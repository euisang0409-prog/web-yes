# 미들웨어 & 데이터베이스 핵심 정리 (시험용)

---

## 1. 미들웨어 (Middleware)

### 개념

* 프로그램과 운영체제, 또는 시스템 사이를 연결해주는 소프트웨어
* 사용자가 직접 보지는 않지만 시스템 동작에 필수

---

### 사용하는 이유

1. **서로 다른 시스템 연결**

   * 언어, 데이터 형식, 통신 방식 차이 해결

2. **공통 기능 제공**

   * 로그인, 보안, 세션, 데이터 변환 등 재사용

3. **시스템 안정성**

   * 요청 분산 처리
   * 트래픽 제어 및 순서 처리

---

### 미들웨어 종류

#### 1) WAS (웹 애플리케이션 서버)

* 동적 웹 처리
* 역할: 비즈니스 로직, DB 연동, 세션, 트랜잭션
* 예: Tomcat, Jeus

---

#### 2) 웹 서버

* 정적 파일 처리 (HTML, CSS, JS, 이미지)
* 예: Apache, Nginx

---

#### 3) DB 미들웨어

* DB와 애플리케이션 연결
* 역할: 연결 관리, SQL 실행, 트랜잭션
* 예: JDBC, MyBatis, Hibernate, JPA

---

#### 4) 메시지 지향 미들웨어

* 시스템 간 메시지 전달
* 예: RabbitMQ, Kafka

---

#### 5) TP 모니터

* 대량 트랜잭션 처리
* 금융/공공기관 사용
* 예: Tuxedo, Tmax, CICS

---

#### 6) 연계 통합 (EAI / ESB)

* 시스템 간 통합
* EAI: 중앙 집중형
* ESB: 분산형 (유연함)

---

#### 7) 인증/보안 미들웨어

* 사용자 인증 및 권한 관리
* 예: Spring Security, OAuth2

---

#### 8) 모니터링 미들웨어

* 시스템 상태 감시
* CPU, 메모리, 응답시간 등
* 예: Scouter

---

## 2. 데이터베이스 기본

### 개념

* **데이터**: 값 (숫자, 문자 등)
* **데이터베이스(DB)**: 데이터를 모아 저장한 것

---

### 특징

* 실시간 접근성
* 계속적 변화
* 동시 공유

---

### DBMS

* 데이터베이스를 관리하는 시스템

---

### DB 유형

* 계층형 (트리)
* 네트워크형 (그래프)
* 관계형 (테이블)
* 객체지향형

---

## 3. 스키마 (중요)

### 1) 외부 스키마

* 사용자 관점
* 필요한 데이터만 표시
* 보안 중요

---

### 2) 개념 스키마

* 전체 구조 정의
* 테이블, 관계, 제약조건 포함

---

### 3) 내부 스키마

* 실제 저장 방식
* 디스크, 인덱스
* 성능 관련

---

## 4. 트랜잭션

* 하나의 작업 단위 (모두 성공 or 모두 실패)

### 명령어

* COMMIT → 저장
* ROLLBACK → 취소 (COMMIT 후 불가)

---

## 5. 관계형 데이터베이스 용어

* 테이블 = 릴레이션
* 행(row) = 튜플
* 열(column) = 속성(attribute)

---

## 6. 데이터 타입

* VARCHAR → 문자열
* INT → 정수

---

## 7. 외래키 (FOREIGN KEY)

### 개념

* 다른 테이블의 값을 참조하는 키

---

### 문법

```sql id="2csm1r"
CONSTRAINT 외래키명 
FOREIGN KEY (컬럼명)
REFERENCES 참조테이블 (컬럼명);
```

---

## 핵심 한 줄 요약

* 미들웨어 = 시스템 연결 및 기능 제공
* 스키마 = DB 구조 정의
* 트랜잭션 = 작업 단위 (COMMIT/ROLLBACK)
* 외래키 = 테이블 간 관계 연결
