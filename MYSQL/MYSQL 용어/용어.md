# SQL 핵심 용어 + 뜻 정리

## 1. 데이터베이스 / 테이블

* **DATABASE** : 데이터를 저장하는 공간 (DB 전체)
* **TABLE** : 데이터를 표 형태로 저장하는 구조
* **COLUMN (컬럼)** : 데이터의 속성 (열)
* **ROW (행)** : 실제 데이터 한 줄

---

## 2. 데이터 타입

* **INT** : 정수 데이터
* **TINYINT** : 작은 정수 (-128 ~ 127)
* **VARCHAR(n)** : 길이가 가변적인 문자열
* **CHAR(n)** : 길이가 고정된 문자열
* **DATETIME** : 날짜 + 시간
* **DATE** : 날짜만

---

## 3. 제약조건 (Constraint)

* **PRIMARY KEY** : 중복 X + NULL X (고유값)
* **NOT NULL** : NULL 값 허용 안 함
* **DEFAULT** : 값 안 넣으면 자동으로 기본값 설정
* **CHECK** : 특정 조건 만족해야 값 입력 가능
* **FOREIGN KEY** : 다른 테이블과 연결하는 키

---

## 4. 제약조건 방식

* **INLINE 방식** : 컬럼 옆에 바로 작성 (간단)
* **TABLE-LEVEL 방식** : CONSTRAINT로 따로 작성 (확장성 좋음)

---

## 5. DDL (데이터 정의어)

* **CREATE TABLE** : 테이블 생성
* **DROP TABLE** : 테이블 삭제
* **ALTER TABLE** : 테이블 구조 수정

---

## 6. ALTER TABLE 관련 키워드

* **ADD COLUMN** : 컬럼 추가
* **DROP COLUMN** : 컬럼 삭제
* **RENAME COLUMN** : 컬럼 이름 변경
* **MODIFY COLUMN** : 컬럼 타입/속성 변경
* **ADD PRIMARY KEY** : 기본키 추가
* **DROP PRIMARY KEY** : 기본키 삭제

---

## 7. 기타 SQL 개념

* **IF NOT EXISTS** : 존재하면 실행 안 함
* **CONSTRAINT** : 제약조건 이름 지정
* **SHOW INDEX** : 인덱스/제약조건 확인
* **KEY_NAME** : 제약조건 이름
* **CURRENT_TIMESTAMP** : 현재 날짜/시간 자동 입력

---

## 8. 주석

* **#** : 한 줄 주석
* **--** : 한 줄 주석
* **/* */** : 여러 줄 주석

## 9. 기타 개념

* **DEFAULT** : 기본값
* **NULL** : 값 없음
* **NOT NULL** : 값 반드시 있어야 함
* **예약어** : SQL에서 의미가 정해진 단어 (백틱 ` 사용)
# DEFAULT / NOT NULL / CHECK 정리 (시험 핵심)

## 1. DEFAULT

* 컬럼에 값이 **아예 입력되지 않았을 때** 자동으로 들어가는 기본값
* 명시적으로 값을 넣지 않을 때만 동작

예:

* 값 생략 → DEFAULT 값 들어감
* NULL 직접 입력 → DEFAULT 적용 안됨

---

## 2. NOT NULL

* 해당 컬럼에 **NULL 값 자체를 허용하지 않음**
* 반드시 값이 들어가야 함

---

## 3. DEFAULT + NOT NULL 같이 사용

* 값 생략 → DEFAULT 값 자동 입력
* NULL 입력 시도 → **에러 발생 (거부)**

즉,

* 입력 안 하면 자동값
* NULL 넣으면 차단

---

## 4. DEFAULT만 있고 NOT NULL 없는 경우

* 값 생략 → DEFAULT 값 들어감
* NULL 직접 입력 → **NULL 그대로 저장됨**

---

## 5. CHECK 제약조건

* 값이 들어올 때 **조건 검사**
* 조건이 TRUE일 때만 허용

예:

* gender IN ('M','W')

---

## 6. CHECK와 NULL (중요)

* NULL은 비교가 안 됨 → 결과 = UNKNOWN
* CHECK는

  * TRUE → 통과
  * FALSE → 거부
  * UNKNOWN → 통과

즉,

* NULL은 CHECK에서 막히지 않음

---

## 7. 핵심 정리

* DEFAULT → 값 없을 때 자동 입력
* NOT NULL → NULL 금지
* CHECK → 조건 검사 (FALSE만 막음)
* NULL → CHECK에서 통과됨 (UNKNOWN)

---

## 한 줄 요약

DEFAULT는 자동값, NOT NULL은 NULL 차단, CHECK는 조건 검사지만 NULL은 막지 못한다.
