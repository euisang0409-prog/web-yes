# 데이터베이스(DB) 전체 정리

---

# 1. 데이터 모델링

## 정의
현실 세계의 업무 내용을 데이터베이스에 저장할 수 있도록 데이터 구조로 설계하는 과정

---

## 모델링 필요한 이유 (이상현상 방지)

1. 데이터 중복 발생

2. 갱신 이상
- 같은 데이터가 여러 곳에 저장되어 일부만 수정 시 불일치 발생

3. 삭제 이상
- 필요한 데이터까지 함께 삭제됨

4. 삽입 이상
- 필요 없는 데이터까지 함께 추가됨

---

# 2. 데이터 모델링 과정

## 1) 요구사항 분석
- 업무에서 관리할 데이터 파악
- 주요 명사 → 엔티티

---

## 2) 개념적 모델링 (ERD)
- 전체 구조를 그림으로 표현
- ERD 사용

### 구성 요소
- 엔티티(Entity)
- 속성(Attribute)
- 관계(Relationship)
- 식별자(Identifier)

---

## 엔티티 관계 예시

- 학생은 수강신청을 한다
- 수업은 수강신청을 받는다
- 강사는 수업을 담당한다

---

## 관계 차수

- 1:1
- 1:N
- M:N → 중간 엔티티로 해결

예:
- 학생 1 : 수강신청 N
- 수업 1 : 수강신청 N
- 학생 M : 수업 N → 수강신청으로 분해

---

# 3. 논리 모델링

- ERD → 테이블 구조 변환
- 테이블 이름 및 속성 정의
- PK, FK 설정
- 정규화 적용

---

# 4. 물리 모델링

- 실제 DBMS에서 실행 가능한 SQL로 변환

설정 내용:
- 컬럼 타입
- NULL 허용 여부
- DEFAULT 값
- PK / FK
- 제약조건

---

# 5. 정규화

## 1NF
- 하나의 속성에는 하나의 값만 존재

---

## 2NF
- 부분 함수 종속 제거

---

## 3NF
- 이행 함수 종속 제거

---

## BCNF
- 모든 결정자는 후보키여야 함

---

## 4NF
- 다중값 종속 제거

---

## 5NF
- 조인으로 원래 테이블 복원 가능

---

# 6. SQL (DML)

## SELECT
SELECT * FROM table;

WHERE 조건 사용

---

## 비교 연산
= 같다
<> != 같지 않다

---

## 논리 연산
AND : 그리고
OR : 또는

---

## NULL 처리
IS NULL
IS NOT NULL

---

## 날짜 비교
hire_date >= '1990-01-01';

---

## LIKE

% : 여러 문자
_ : 한 문자

'A%' : A로 시작
'%A' : A로 끝
'%A%' : A 포함
'A%B' : A로 시작하고 B로 끝

'A_' : A로 시작 + 2글자
'A__' : A로 시작 + 3글자
'_A' : A로 끝 + 2글자
'_A_' : A가 가운데 + 3글자

---

## INSERT
INSERT INTO table (col1, col2)
VALUES (val1, val2);

---

## UPDATE
UPDATE table
SET col = value
WHERE 조건;

---

## DELETE
DELETE FROM table
WHERE 조건;

---

# 7. DCL (권한 관리)

## 개념
데이터베이스 사용자 권한 관리

---

## 사용자 생성
CREATE USER 'user'@'localhost'
IDENTIFIED BY '1234';

---

## 권한 부여
GRANT SELECT
ON db.table
TO 'user'@'localhost';

---

## 권한 회수
REVOKE SELECT
ON db.table
FROM 'user'@'localhost';

---

## 사용자 확인
SELECT user, host FROM mysql.user;

---

## 사용자 삭제
DROP USER 'user'@'localhost';

---

## 권한 종류
SELECT, INSERT, UPDATE, DELETE
CREATE, DROP, ALTER
INDEX
REFERENCES
ALL PRIVILEGES

---

# 8. TCL (트랜잭션)

## 개념

---

## 예시
UPDATE account SET balance = balance - 1000 WHERE id = 1;
UPDATE account SET balance = balance + 1000 WHERE id = 2;
