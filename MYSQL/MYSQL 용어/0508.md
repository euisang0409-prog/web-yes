# SQL 정리 (DDL / DML / Stored Program)

---

# INDEX

## 개념

- 조회 속도를 빠르게 하기 위한 구조
- WHERE / ORDER BY / GROUP BY 자주 쓰는 컬럼에 사용
- 미리 데이터 위치를 기록해 빠르게 검색
- INSERT / UPDATE / DELETE는 성능 저하 가능

## 생성

CREATE INDEX 인덱스명 ON 테이블(컬럼);

## 조회

SHOW INDEX FROM 테이블명;

## 삭제

DROP INDEX 인덱스명 ON 테이블명;

# VIEW

## 개념

- SELECT 결과를 가상 테이블로 저장
- 실제 데이터 저장 X
- 원본 데이터 변경 시 VIEW도 변경됨
- 복잡한 SELECT문 단순화

## 특징

- VIEW 안에 VIEW 가능 (성능 저하 가능)
- JOIN 없이 복잡한 SELECT 단순화 가능

# SUBQUERY

## 개념

SQL 안에 SQL이 들어가는 구조

## 사용 상황

- 조건을 다른 테이블 결과로 판단
- 평균 / 최대값 기준 조회
- JOIN 없이 단계적 처리

## 종류

- 단일 행 서브쿼리 (결과 1개)
- 다중 행 서브쿼리 (IN / ANY / ALL)
- 다중 컬럼 서브쿼리
- 상관 서브쿼리
- FROM 서브쿼리

## 예시

SELECT emp_name
FROM employee
WHERE department = (
SELECT department
FROM employee
WHERE salary >= 6000
);

# JOIN

## 개념

정규화된 테이블을 PK / FK 기준으로 연결

## 기본 구조

SELECT 컬럼
FROM A a
JOIN B b
ON a.id = b.id;

## 종류

- INNER JOIN : 일치 데이터
- LEFT JOIN : 왼쪽 전체 + 오른쪽 매칭
- RIGHT JOIN : 오른쪽 전체 + 왼쪽 매칭
- CROSS JOIN : 모든 조합

## ON vs WHERE

- ON : 조인 조건
- WHERE : 결과 필터링

# AUTO_INCREMENT

## 개념

자동으로 숫자 증가 (PK와 함께 사용)

## 특징

- 삭제해도 재사용 안함
- ROLLBACK해도 증가 유지
- 고유 식별자 용도

## 설정

ALTER TABLE 테이블명 AUTO_INCREMENT = 1000;

## 확인

SHOW TABLE STATUS LIKE '테이블명';
SELECT LAST_INSERT_ID();

# LIMIT

## 개념

조회 결과 개수 제한

SELECT \* FROM 테이블 LIMIT 10;

## 페이징

LIMIT 0, 10
LIMIT 10, 10

# STORED PROGRAM

## 개념

SQL을 프로그래밍처럼 실행하는 기능

## 종류

- Stored Procedure
- Stored Function
- Trigger
- Event

## 기본 구조

DELIMITER //
CREATE PROCEDURE hello()
BEGIN
SELECT 'Hello';
END //
DELIMITER ;
