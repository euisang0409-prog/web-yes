# INDEX & SUBQUERY 정리

---

# INDEX

## INDEX란?
- 데이터베이스에서 데이터를 빠르게 찾기 위한 자료구조
- 테이블 전체를 전부 조회하는 것을 **Full Table Scan**이라고 한다.
- 특정 컬럼에 INDEX를 생성하면 해당 위치를 빠르게 탐색할 수 있다.

---

# INDEX를 사용하는 이유

- 데이터가 많은 테이블에서 SELECT 속도를 빠르게 하기 위해 사용
- 데이터가 적을 때는 차이가 거의 없지만,
  데이터가 많아질수록 성능 차이가 매우 커진다.

---

# INDEX 생성 문법

```sql
CREATE INDEX 인덱스명
ON 테이블명(컬럼명);
```

## 예시

```sql
CREATE INDEX idx_employee_department
ON employee(department);
```

---

# INDEX 확인

```sql
SHOW INDEX FROM 테이블명;
```

## 예시

```sql
SHOW INDEX FROM employee;
```

---

# INDEX 삭제

```sql
DROP INDEX 인덱스명
ON 테이블명;
```

## 예시

```sql
DROP INDEX idx_employee_department
ON employee;
```

---

# 자동으로 INDEX가 생성되는 경우

## PRIMARY KEY

```sql
CREATE TABLE student(
    id INT PRIMARY KEY
);
```

- PRIMARY KEY 지정 시 자동으로 INDEX 생성

---

## UNIQUE

```sql
CREATE TABLE member(
    email VARCHAR(100) UNIQUE
);
```

- UNIQUE 제약조건도 자동으로 INDEX 생성

---

# INDEX의 단점

1. 별도의 저장공간 필요
2. INSERT / UPDATE / DELETE 작업 시 성능이 느려질 수 있음

이유:
- 데이터를 수정할 때 INDEX도 같이 수정해야 하기 때문

---

# INDEX를 걸면 좋은 컬럼

## 1. WHERE 조건에 자주 사용하는 컬럼

```sql
SELECT *
FROM employee
WHERE department = '개발';
```

---

## 2. JOIN에 자주 사용하는 컬럼

```sql
SELECT *
FROM orders o
INNER JOIN customer c
ON o.customer_id = c.customer_id;
```

---

## 3. ORDER BY에 자주 사용하는 컬럼

```sql
SELECT *
FROM employee
ORDER BY salary DESC;
```

---

## 4. GROUP BY에 자주 사용하는 컬럼

```sql
SELECT department, AVG(salary)
FROM employee
GROUP BY department;
```

---

# 단일 컬럼 INDEX

- 하나의 컬럼에 생성하는 INDEX

```sql
CREATE INDEX idx_employee_department
ON employee(department);
```

---

# 복합 INDEX

- 여러 컬럼을 묶어서 생성하는 INDEX

```sql
CREATE INDEX idx_employee_dept_salary
ON employee(department, salary);
```

---

# INDEX 사용 여부 확인

- SQL 문장이 INDEX를 사용하는지 확인 가능

```sql
EXPLAIN SELECT *
FROM employee
WHERE department = '개발';
```

- 실행 계획을 보여준다.
- INDEX 사용 여부 확인 가능

# SUBQUERY

## SUBQUERY란?

- SQL 안에 또 다른 SQL을 넣어서 사용하는 기법
- 메인쿼리(Main Query) 안에 포함되는 SELECT문
- 괄호 안에서 먼저 실행된 후 결과를 메인쿼리가 사용한다.

---

# SUBQUERY 사용 이유

## 1. 특정 조건을 찾기 어려울 때

```sql
SELECT *
FROM employee
WHERE salary > (
    SELECT AVG(salary)
    FROM employee
);
```

---

## 2. 집계 결과를 조건으로 사용하고 싶을 때

```sql
SELECT department, AVG(salary)
FROM employee
GROUP BY department
HAVING AVG(salary) > (
    SELECT AVG(salary)
    FROM employee
);
```

---

## 3. 존재 여부를 판단할 때

```sql
SELECT *
FROM customer
WHERE EXISTS (
    SELECT *
    FROM orders
    WHERE customer.customer_id = orders.customer_id
);
```

---

# 다중 컬럼 SUBQUERY

## 사용하는 이유

- 단일 컬럼 비교만으로는 정확한 비교가 어려울 수 있다.
- 여러 컬럼을 하나의 세트처럼 비교하기 위해 사용

---

# 핵심 개념

```sql
(컬럼1, 컬럼2)
```

- 두 컬럼을 하나의 세트로 비교
- 컬럼 간의 관계를 유지 가능

---

# 예시

```sql
SELECT *
FROM employee
WHERE (department, salary) IN (
    SELECT department, MAX(salary)
    FROM employee
    GROUP BY department
);
```

의미:
- 부서별 최고 급여 직원 조회

# INLINE VIEW

## FROM 절에서 사용하는 SUBQUERY

- 서브쿼리 결과를 임시 테이블처럼 사용

---

# 문법

```sql
SELECT *
FROM (
    SELECT ...
) t;
```

---

# 예시

```sql
SELECT *
FROM (
    SELECT department,
           AVG(salary) AS avg_salary
    FROM employee
    GROUP BY department
) t
WHERE avg_salary >= 5000;
```

---

# INLINE VIEW를 사용하는 이유

1. 중간 결과를 재사용 가능
2. 복잡한 통계 처리 가능
3. 정렬 및 필터링이 쉬움
4. 가독성이 좋아짐

---

|---|---|
| INDEX | 검색 속도 향상 |

| 복합 INDEX | 여러 컬럼을 묶은 INDEX |
| 다중컬럼 SUBQUERY | 컬럼 관계를 유지하며 비교 |
