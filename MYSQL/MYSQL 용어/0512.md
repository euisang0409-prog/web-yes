# CASE문
- 조건에 따라 다른 값을 반환하는 문법

CASE
    WHEN 조건1 THEN 결과1
    WHEN 조건2 THEN 결과2
    WHEN 조건3 THEN 결과3
    ELSE 기본결과
END

- CASE문은 SELECT 절에서 많이 사용된다
- ORDER BY 절에서도 사용할 수 있다
  -> 비즈니스 로직 기준 정렬
  -> 업무 우선순위
  -> 처리 순서
  -> 긴급도
- GROUP BY에서도 사용할 수 있다
- UPDATE에서도 조건에 맞게 데이터를 수정할 수 있다

------------------------------------------------------------

# 반복문
- 같은 작업을 여러 번 반복 실행할 때 사용
- 프로시저 안에서만 작성 가능

------------------------------------------------------------

# WHILE

BEGIN

    변수 선언;

    WHILE 조건 DO

        반복 실행할 문장;

        SET 변수 = 값;

    END WHILE;

END

- 조건이 TRUE인 동안 반복 실행

------------------------------------------------------------

# REPEAT
- do-while과 사용 방식이 비슷

REPEAT

    반복 실행할 명령;

UNTIL 조건

END REPEAT;

- 먼저 실행 후 조건 검사
- 조건이 TRUE가 되면 종료

------------------------------------------------------------

# LOOP
- 무한루프 기반
- 종료 조건을 직접 작성해야 함
- 라벨 필요

라벨명: LOOP

    반복 실행할 문장;

    IF 조건 THEN
        LEAVE 라벨명;
    END IF;

END LOOP;

- 가장 자유로운 반복문
- 종료 조건 반드시 필요

------------------------------------------------------------

# ITERATE
- continue 역할
- 현재 반복 중단 후 다음 반복으로 이동

라벨명: LOOP

    IF 조건 THEN
        ITERATE 라벨명;
    END IF;

END LOOP;

------------------------------------------------------------

# MySQL 예외처리 (Handler)

- Procedure 안에서 에러, 경고, 조회 결과 없음 등을 처리하는 예외처리 문법
- 프로시저 실행 중 문제가 생겨도 바로 종료되지 않도록 처리 가능

------------------------------------------------------------

# Handler 사용 예시

에러 처리
- 중복 PK INSERT

조회 결과 없음 처리
- SELECT ... INTO 결과 없음

커서 종료 처리
- 더 이상 읽을 데이터 없음

경고 처리
- 데이터 잘림
- 형 변환 경고

------------------------------------------------------------

# CURSOR
- SELECT 결과를 한 줄씩 꺼내서 처리하는 기능

------------------------------------------------------------

# Handler 기본 문법

DECLARE 핸들러타입 HANDLER FOR 예외

BEGIN

    예외 처리 문장;

END;

------------------------------------------------------------

# 핸들러 타입

CONTINUE
- 에러 처리 후 다음 문장 계속 실행

EXIT
- 에러 처리 후 BEGIN-END 블록 종료

------------------------------------------------------------

# 예외 종류

SQLEXCEPTION
- 대부분의 SQL 에러

SQLWARNING
- SQL 경고

NOT FOUND
- 조회 결과 없음

1062
- 중복 키 에러

------------------------------------------------------------

# Trigger

- INSERT, UPDATE, DELETE 발생 시 자동 실행되는 SQL 프로그램

------------------------------------------------------------

# Trigger 사용 예시

- 주문 발생 시 재고 자동 차감
- 회원 수정 시 수정 로그 저장
- 급여 음수 입력 방지
- 게시글 삭제 기록 저장

------------------------------------------------------------

# 프로시저 vs 트리거

프로시저
- 직접 호출해야 실행

트리거
- 자동 실행

------------------------------------------------------------

# Trigger 실행 시점

BEFORE
- 저장, 수정, 삭제 전에 실행
- 검사 목적

AFTER
- 저장, 수정, 삭제 후 실행
- 기록, 로그 저장 목적

------------------------------------------------------------

# Trigger 이벤트 종류

INSERT
- 데이터 추가 시 실행

UPDATE
- 데이터 수정 시 실행

DELETE
- 데이터 삭제 시 실행

------------------------------------------------------------

# NEW / OLD

NEW
- 새로 입력되거나 수정될 값

OLD
- 기존 값

------------------------------------------------------------

# Trigger 기본 문법

DELIMITER //

CREATE TRIGGER 트리거명

[BEFORE 또는 AFTER]
[INSERT 또는 UPDATE 또는 DELETE]

ON 테이블명

FOR EACH ROW

BEGIN

    실행할 SQL;

END //

DELIMITER ;
