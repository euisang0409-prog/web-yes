# MySQL 기반 데이터베이스 정리

## 1. 스키마 (Schema)

스키마 = 데이터베이스
데이터를 어떻게 저장할 것인지에 대한 설계도이다.

### 스키마의 구성 요소

* 테이블 (Table)
* 뷰 (View)
* 인덱스 (Index)
* 트리거 (Trigger)
* 저장 프로시저 (Procedure)

### 스키마를 사용하는 이유

1. 데이터 구조를 정리하기 위해

* 구조를 정해서 일관성을 유지

2. 데이터 관리 단위로 사용하기 위해

* 프로젝트별로 DB를 나눌 수 있음
* 예: 쇼핑몰 / 게시판 / 회원관리

3. 접근 권한 관리

* 특정 스키마만 접근하도록 설정 가능

## 2. 데이터베이스 3계층 구조

### 1) 외부 스키마

* 화면에 보이는 데이터

### 2) 개념 스키마

* 전체 데이터 구조

### 3) 내부 스키마

* 실제 저장 방식
* 디스크에 어떻게 저장되는지

## 3. 기본 개념

### Data

* 가공되지 않은 단순한 사실이나 값

### DB (Database)

* 데이터를 체계적으로 저장해놓은 공간

### DBMS

* 데이터베이스를 관리하는 프로그램

---

## 4. DBMS 종류

### 1) 계층형 데이터베이스

* 트리 구조
* 1:N 관계
* 구조 단순
* 변경 어려움

### 2) 네트워크형 데이터베이스

* 그래프 구조
* N:N 관계 가능
* 구조 복잡

### 3) 관계형 데이터베이스 (RDBMS)

* 테이블 구조
* 가장 많이 사용
* SQL 사용
* 무결성 보장

### 4) 객체지향 데이터베이스

* 객체 형태 저장
* Java/C++ 객체 저장 가능
* 사용 빈도 낮음

### 5) NoSQL

* 관계형이 아닌 DB

종류:

* Key-Value
* Document (JSON)
* Column
* Graph

특징:

* 대용량 처리
* 빠른 속도
* 유연한 구조

---

## 5. SQL (Structured Query Language)

데이터를 읽고, 쓰고, 삭제하는 등 DB를 관리하는 언어

---

## 6. DDL (Data Definition Language)

데이터베이스 구조 정의

* CREATE: 생성
* DROP: 삭제
* ALTER: 수정
* TRUNCATE: 테이블 유지, 데이터 전체 삭제

---

## 7. 데이터베이스 생성

```sql
CREATE DATABASE IF NOT EXISTS db_name
CHARACTER SET utf8mb4
COLLATE utf8mb4_unicode_ci;
```

* CHARACTER SET: 문자 인코딩 설정
* utf8mb4: MySQL 표준 유니코드
* COLLATE: 정렬 및 비교 기준

---

## 8. 테이블 구조

* 행(Row): 레코드, 튜플
* 열(Column): 속성
* 카디널리티: 행 개수

## 10. 제약조건

### PRIMARY KEY

* 유일 + NULL 불가

### NOT NULL

* 반드시 값 존재

### UNIQUE

* 중복 불가

### FOREIGN KEY

* 다른 테이블 PK 참조

### CHECK

* 조건 제한

### DEFAULT

* 기본값 설정

---

## 11. 테이블 생성

```sql
CREATE TABLE table_name (
    column_name datatype
);
```

---

## 12. ALTER (테이블 수정)

### 컬럼 추가

```sql
ALTER TABLE table_name ADD COLUMN column_name datatype;
```

### 컬럼 삭제

```sql
ALTER TABLE table_name DROP COLUMN column_name;
```

### 컬럼 이름 변경

```sql
ALTER TABLE table_name RENAME COLUMN old_name TO new_name;
```

### 자료형 변경

```sql
ALTER TABLE table_name MODIFY COLUMN column_name datatype;
```

---

## 13. 무결성

### 1) 개체 무결성

* PK는 NULL 불가 + 유일

### 2) 참조 무결성

* FK는 NULL 또는 참조 테이블 값만 가능

### 3) 도메인 무결성

* 값의 범위 제한

### 4) 사용자 정의 무결성

* 비즈니스 규칙 적용
* 예: 나이 음수 불가
