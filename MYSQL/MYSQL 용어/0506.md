# SQL 정리 (집계 함수 / GROUP BY / HAVING / ORDER BY / INDEX)

## 1. 집계 함수 (그룹 함수)

- 하나 이상의 행을 하나의 값으로 집계하는 함수
- COUNT(개수), SUM(합계), AVG(평균), MAX(최대값), MIN(최소값)
- 일반 컬럼과 집계 함수는 단독 SELECT에서 함께 사용 불가 (GROUP BY 필요)
- NULL 값은 집계에서 제외됨

SELECT AVG(salary)
FROM employees;

---

## 2. GROUP BY

- 같은 값을 기준으로 데이터를 그룹화하여 집계

SELECT department_id, AVG(salary)
FROM employees
GROUP BY department_id;

- 여러 컬럼 기준으로 그룹화 가능

---

## 3. HAVING

- GROUP BY로 만들어진 그룹에 조건을 거는 문법
- WHERE: 행 단위 필터링 (그룹 전)
- HAVING: 그룹 단위 필터링 (그룹 후)

SELECT department_id, AVG(salary)
FROM employees
GROUP BY department_id
HAVING AVG(salary) > 3000;

---

## 실행 순서

SELECT → FROM → WHERE → GROUP BY → HAVING → ORDER BY

---

## 4. ORDER BY (정렬)

- 조회 결과를 정렬하는 문법
- ASC: 오름차순 (작은 → 큰)
- DESC: 내림차순 (큰 → 작은)

SELECT \*
FROM students
ORDER BY age ASC;

- 다중 정렬

SELECT \*
FROM students
ORDER BY age DESC, name ASC;

- 정렬 기준
  숫자: 작은 → 큰
  문자: ㄱ → ㅎ / A → Z
  날짜: 과거 → 최신

---

## 5. 순위 함수 (윈도우 함수)

- ROW_NUMBER(): 순차 번호
- RANK(): 공동 순위 + 다음 순위 건너뜀
- DENSE_RANK(): 공동 순위 + 순위 안 건너뜀
- NTILE(n): n개 그룹으로 나눔
- PERCENT_RANK(): 상대적 순위 비율

RANK() OVER (ORDER BY salary DESC)

---

## 6. INDEX (인덱스)

- 데이터를 빠르게 찾기 위한 검색용 구조 (목차 역할)
- Full Table Scan 대신 빠른 조회 가능

SELECT \* FROM student WHERE name = '박민수';

---

## 인덱스 생성

CREATE INDEX idx_student_name
ON student(name);

## 인덱스 확인

SHOW INDEX FROM student;

## 인덱스 삭제

DROP INDEX idx_student_name ON student;
