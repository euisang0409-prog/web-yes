index
- 데이터베이스에서 데이터를 빠르게 찾기 위한 자료구조
- 테이블 전체에서 찾는것을 Full Table Scan이라고 한다.
- 특정 속성을 index로 만들어 놓으면 해당 위치를 빠르게 찾아간다.

인덱스를 사용하는 이유
- 데이터가 많은 테이블에서 select를 빠르게 하기 위해 사용한다.
- 데이터가 적을 때는 차이가 거의 없지만 많아지면 차이가 엄청 커진다.

인덱스를 생성하는 문법
CREATE INDEX 인덱스명 ON 테이블명(컬럼);

인덱스 확인
SHOW INDEX FROM 테이블명;

인덱스 삭제
DROP INDEX 인덱스명 ON 테이블명;

테이블 생성시 PK를 지정하면 자동으로 INDEX가 만들어진다.
UNIQUE 제약조건도 자동으로 인덱스를 만들어준다.

단점
- 별도의 저장공간이 필요하다.
- INSERT ,UPDATE,DELETE 작업에서는 느려질 수 있다.

인덱스를 걸면 좋은 컬럼
1. WHERE 조건에 자주 사용되는 컬럼
2. JOIN에 자주 사용되는 컬럼
3. ORDER BY에 자주 사용되는 컬럼
4. GROUP BY에 자주 사용되는 컬럼

단일 컬럼 인덱스
- 하나의 컬럼에 만드는 인덱스
CREATE INDEX idx_employee_department ON employee(department)

복합 인덱스
- 여러 컬럼을 묶어서 만드는 인덱스
CREATE INDEX idx_employee_dept_salary
ON employee(department,salary);

내가 작성한 SELECT 문장이 INDEX를 사용하는지 확인하는 방법
문장앞에 EXPLAIN 작성하기

SubQuery
- SQL안에 또 다른 SQL을 넣어 사용하는 기법
- 메인쿼리(Main Query)안에 포함되는 부속 SELECT문
- 괄호 안에서 실행되며, 먼저 실행한 뒤 그 결과를 메인 쿼리가 사용한다.

서브쿼리가 사용되는 경우
- 특정 조건을 찾기 어려울때
- 집계 결과를 다시 조건으로 사용하고 싶을 때
- 존재 여부를 판단할 때

다중컬럼 서브쿼리를 사용하는 이유
- 단순 단일컬럼 비교로는 문제가 생길 수 있다.

다중컬럼 비교의 핵심 
(컬럼1,컬럼2)를 하나의 세트로 비교한다.
두 컬럼의 관계까지 유지

FROM 절에서 사용되는 서브쿼리(Inline view)
- 서브쿼리로 조회된 값을 테이블 처럼 사용한다.
