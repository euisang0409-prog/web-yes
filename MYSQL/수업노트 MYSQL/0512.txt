CASE문
- 조건에 따라 다른 값을 반환하는 문법

CASE
	WHEN 조건1 THEN 결과1
	WHEN 조건2 THEN 결과2
	WHEN 조건3 THEN 결과3
	else 기본결과
END

- CASE문은 SELECT 절에서 많이 사용된다
- order by절에서도 사용할 수 있다. -> 비즈니스 로직 기준 정렬(업무 우선순위, 처리 순서, 긴급도)
- gorup by에서도 사용할 수 있다.
- update 쪽에서도 조건에 맞게 데이터를 수정할 수 있는 장점

반복문
- 같은 작업을 여러번 반복 실행할 때
- 프로시저 안에서만 작성 가능

WHILE
begin
변수 선언
  while 조건 do
	반복실행할 문장
	set 으로 변수값 변경
  end while
end

REPEAT : do -while과 사용법이 같다.
repeat
	반복실행할 명령;

until 조건; -> 조건이 참이 되면 종료
end repeat

LOOP : 무한루프 기반, 종료조건을 명시해야 한다.
- 라벨을 달아야 한다.

라벨 : loop
	반복 실행할 문장;
	if 조건 then
	     leave 라벨
	end if;
end loop;				0511 복습내용

ITERATE -> contiue
- 현재 반복을 중단하고 다음 반복으로 바로 이동하는 명령어

라벨 : loop
	if 조건 then
		interate 라벨;
	end if;
end loop;				0511 복습

mySQL 예외처리
Handler
- Procedure 안에서 에러,  경고, 조회 결과 없음 같은 상황을 처리하는 예외처리 문법
- 프로시저 실행중 문제가 생겼을 때 프로시저가 바로 중단되지 않도록 처리 방법을 정해두는 문법

에러 처리 : 중복된 PK를 INSERT한 경우
조회 결과 없음 처리 : SELECT ... INTO의 결과가 없는 경우
커서 종료 처리 : 더 이상 읽을 데이터가 없는 경우
ㄴ CURSOR : SELECT 결과를 한 줄씩 꺼내서 처리하는 기능
경고 처리 : 데이터 잘림, 형 변환 경고 등

기본 문법
DECLARE 핸들러타입 HANDLER FOR 예외
BEGIN
예외처리


END;

핸들러 타입 종류
CONTINUE : 에러 처리 후 다음 문장 계속 실행
EXIT : 에러 처리 후 BEGIN - END 블록 종료

예외종류
SQLECEOTION : 대부분의 SQL 에러
SQLWARNING : SQL 경고
NOT FOUND : 조회 결과 없음
MySQl 에러코드 : 1062 (중복 키 에러) 

-- Trigger 
-- insert, update, delete가 발생했을 때 실행되는 sql 프로그램
-- 주문이 들어오면 재동 자동 차감
-- 회원 정보가 수정되면 수정 로그를 저장 
-- 급여가 음수가 입력되면 막기
-- 게시글이 삭제되면 기록 남기기

-- 프로시저는 직접 호출해야 실행 된다.
-- 트리거는 자동으로 실행된다.

-- 트리거 작동 시점
-- BEFORE : 데이터가 저장, 수정,삭제 되기 전에 실행(검사)
-- AFTER : 데이터가 저장,수정,삭제 된 후 실행(기록, 다른테이블을 수정)

-- 실행 이벤트
-- INSERT : 데이터 추가시 실행
-- UPDATE : 데이터 수정시 실행
-- DELETE : 데이터 삭제시 실행

-- NEW : 새로 들어오거나 수정될 값
-- OLD : 기존에 있던 값

-- 기본 문법
-- DELIMITER // 
-- CREATE TRIGGER 트리거명
-- [BEFORE 또는 AFTER] [INSERT 또는 UPDATE 또는 DELETE]
-- ON 테이블명
-- FOR EACH ROW
-- BEGIN
-- 실행할 SQL;
-- END //
-- DELIMITER ;
