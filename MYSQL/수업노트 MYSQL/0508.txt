Sub Query

- join없이 단계적으로 데이터를 조회하고 싶을 때

종류
- 단일 행 서브쿼리 : 결과가 1개
- 다중 행 서브쿼리 : 결과가 여러개

IN : 여러 값 중 하나

- 다중 컬럼 서브쿼리 : 컬럼이 여러개
- 상관 서브쿼리 : 바깥 쿼리와 연결됨
- from절 서브쿼리 : 가상 테이블처럼 사용

Join
- 정규화를 통해서 분리된 테이블 에서 공통된 컬럼을 기준으로 연결해서
조회하는 기능

PK와 FK로 연결된 컬럼이 있어야 한다.


INNER JOIN
- 두 테이블에서 조건이 일치하는 데이터만 조회

LEFT OUTER JOIN
- 왼쪽 테이블(FROM절에 작성된 테이블)의 데이터는 전부 조회하고,
 오른쪽 테이블(JOIN된 테이블)의 데이터는 조건이 맞는 데이터만 조회
- 없는 데이터 쪽은 NULL로 표시한다.

RIGHT OUTER JOIN
- 오른쪽 테이블은 전부 조회하고, 왼쪽 테이블은 조건이 맞는 데이터만 조회
- LEFT와 RIGHT는 방향만 바뀐것이기 때문에 테이블 순서를 바꿔서 LEFT를 많이 사용한다.

CROSS JOIN
- 두 테이블에 들어있는 데이터의 모든 조합을 만든다.

핵심문법
SELECT 조회하고싶은 컬럼
FROM 기준 테이블 별칭
JOIN 연결 테이블 별칭
ON 기준테이블.컬럼 = 연결테이블.컬럼;

ON절이 두 테이블을 연결하는 기준

외래키가 없어도 조인은 가능하나 무결성을 해칠 수 있기 때문에 권장하지 않는다. -> 시험 나옴

AUTO_INCREMENT
- 숫자 값을 자동으로 1씩 증가시켜서 입력해주는 속성
- 데이터를 추가할 때마다 직접 번호를 입력하면 번거롭기 때문에 사용을한다.
- 반드시 숫자 타입의 컬럼에만 붙여야 한다.
- 인덱스가 있어야 하기 때문에 pk와 함께 사용한다.
- 한 테이블에 하나만 사용할 수 있다.
- 직접 값을 넣을 수 도 있다.
INSERT INTO board(board_id, tilte, content)
values(100, '관리자 공지', '직접 번호 입력');
그 다음 자동 증가 값은 보통 101부터 시작한다.
- 중간 번호는 재사용하지 않는다.
1. 	첫번째글
2	두번째글
3	세번째글

delete from board
where board_id = 3;

새 글을 추가하게 되면 board_id가 4로 추가된다.
- AUTO_INCREMENT는 연속 번호 보장용이 아니라 고유 식별자 생성용이다.

- INSERT 했다가 ROLLBACK 해도 다음 번호로 건너뛴다.
- 테이블 생성 후 다음 자동 증가값을 바꿀 수 있다.
ALTER TABLE 테이블명 AUTO_INCREMENT = 1000;
이후 새 데이터는 1000부터 들어갈 수 있다.

- 현재 AUTO_INCREMENT 값 확인하기
SHOW TABLE status LIKE '테이블명';

- 마지막으로 생성된 번호 확인하기
select LAST_INSERT_ID();

LIMIT
- 조회 결과를 제한하는 절
- 몇개까지만 보여줘

SELECT *
FROM 테이블
LIMIT 개수;

ORDER BY와 함께 많이 사용된다.
내림차순으로 정렬해서 상위 5개만 조회해라

데이터가 몇백만건 들어있는 테이블에서 LIMIT이 없으면 위험할 수 있다.
LIMIT 시작위치,개수
LIMIT 개수 OFFSET 시작위치  

페이징처리할 때 사용
SELECT *
ORDER BY BOARD_ID DESC
LIMIT 0, 10, 0개 건너뛰고 10개 조회

SELECT *
ORDER BY BOARD_ID DESC
LIMIT 10, 10 10개 건너뛰고 10개 조회

Stored Program의 핵심 구조
DELIMITER // 
CREATE PROCEDURE hello()
BEGIN
	SELECT 'Hello MySQL Stored Program' AS message;

매개변수 IN, OUT, INOUT

IN은 외부에서 값을 받아오는 매개변수
CREATE PROCEDURE 프로시저명(IN 매개변수명 타입)

OUT은 프로시저 내부에서 만든 값을 외부로 내보내는 매개변수
CREATE PROCEDURE 프로시저명(out매개변수명 타입)

INOUT은 값을 받아오고, 수정한뒤 다시 내보내는 역할
CREATE PROCEDURE 프로시저명(INOUT 매개변수명 타입)
