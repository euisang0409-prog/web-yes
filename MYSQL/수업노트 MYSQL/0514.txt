실습 시나리오

한 학원에서 기존 수강 관리 시스템의 문제점을 해결하기 위해 새로운 시스템 구축 프로젝트를 진행하려고 한다.

현재 시스템에는 다음과 같은 문제가 존재한다.

- 모바일 사용이 불편하다.
- 출결 처리 속도가 느리다.
- 문자 발송 실패가 자주 발생한다.
- 상담 이력이 통합 관리되지 않는다.
- 관리자마다 사용하는 프로그램이 다르다.
- 수강료 결제 내역이 자주 누락된다.

여러분은 요구사항 확인 단계에서 현행 시스템을 분석하고, 요구사항을 정리하는 업무를 담당한다.

현재 회사는 다음 목표를 가지고 시스템 개선을 진행하려고 한다.

- 기존 핵심 기능은 최대한 유지한다.
- 문제가 발생하는 기능은 개선한다.
- 사용자 편의 기능을 새롭게 추가한다.

1. 유지해야 하는 기능 2개 작성

- 현재 정상적으로 동작해야 한다.
- 사용자 이용 빈도가 높은 기능이어야 한다.
- 문제점이 제시된 기능은 작성할 수 없다.
ㄴ 회원가입 : 현재 정상적으로 동작하며 사용자가 서비스를 이용하기 위해 반드시 필요한 기능
ㄴ장바구니 담기 : 현재 정상적으로 동작하며 상품 구매 과정에서 자주 사용하는 핵심 기능이다.


2. 개선해야 하는 기능 3개 작성

- 현재 문제점이 존재하는 기능만 작성한다.
- 문제 원인 또는 현재 문제 상황을 함께 작성한다.
- 어떤 방향으로 개선할 것인지 함께 작성한다.

작성 형식
- 기능명 	:  상품 조회 기능
- 현재 문제 :  검색 속도가 느림
- 개선 방향 : 데이터베이스 검색 최적화 및 INDEX를 적용하여 조회 속도를 개선한다.

작성 예시
배송 조회 기능
- 현재 문제: 모바일 화면이 깨져서 사용하기 어렵다.
- 개선 방향: 반응형 UI를 적용하여 모바일 환경에서도 정상적으로 보이도록 수정한다.

3. 새롭게 추가해야 하는 기능 2개 작성

- 현재 시스템에 존재하지 않는 기능이어야 한다.
- 실제 쇼핑몰에서 사용할 수 있는 현실적인 기능이어야 한다.
- 기존 문제를 해결하거나 사용자 편의성을 높일 수 있어야 한다.

1. 상품 리뷰 기능
사용자가 상품 구매 후 리뷰와 평점을 작성할 수 있도록 추가한다.
상품 선택에 도움을 주어 사용자 편의성을 높일 수 있다.

2. 찜하기(관심상품) 기능
사용자가 관심 있는 상품을 저장할 수 있도록 추가한다.
나중에 다시 쉽게 확인할 수 있어 구매 편의성을 높일 수 있다.
