## 1. 패키지 관리자

리눅스는 텍스트 기반 환경이기 때문에 사용자가 직접 웹에서 프로그램을 다운로드하기보다 패키지 관리자를 통해 저장소(repository)에 접근하여 프로그램을 설치한다.

```bash
sudo apt install 프로그램명
```

---

## 2. 배포 (Deployment)

### 로컬 환경

* 내 컴퓨터에서만 실행됨
* 외부 사용자가 접속 불가
* IP/포트가 외부에 노출되지 않음

### 문제점

* 다른 사람이 서비스 사용 불가

---

### 클라우드 배포

* AWS 등 서버를 빌려 서비스 운영
* 외부에서 접속 가능

구조:

* 서버에 프로그램 업로드
* 서버 IP를 통해 접속

주의:

* 외부 공개로 보안 위험 존재
* 네트워크 설정 필요

---

## 3. 웹 서버 (WAS)

웹 서버는 사용자의 요청을 받아 서버로 전달하는 역할을 한다.

* Nginx: 대표적인 웹 서버

```bash
sudo apt install nginx
```

```bash

```

```bash

```

## 7. 파일 권한

구성:

* user (소유자)
* group (그룹)
* other (기타 사용자)

권한:

* r (read) = 4
* w (write) = 2
* x (execute) = 1

예시:

```bash
rwxr--r--
```

숫자 표현:

```bash
chmod 644 파일명
```



## 11. OSI 7계층

네트워크 통신을 7단계로 나눈 구조

### 필요한 이유

* 통신 구조를 쉽게 이해
* 문제 발생 시 원인 파악
* 장비 역할 구분 가능

---

## 12. 각 계층 정리

### 1계층: 물리 계층 (Physical Layer)

* 전기/신호로 데이터 전달
* 장비: 허브, 리피터
* 특징: 데이터 의미 모름 (0과 1)

---

### 2계층: 데이터 링크 계층 (Data Link Layer)

* 같은 네트워크 내 전달
* MAC 주소 사용
* 장비: 스위치

---

### 3계층: 네트워크 계층 (Network Layer)

* IP 주소 기반 경로 결정
* 장비: 라우터

---

### 네트워크 종류

* LAN: 집, 학교, 회사
* MAN: 도시 단위
* WAN: 국가/인터넷

---

### 4계층: 전송 계층 (Transport Layer)

* TCP / UDP
* 데이터 분할, 순서, 재전송
* 포트 번호 사용

예:

* 80: HTTP
* 443: HTTPS

---

### 5계층: 세션 계층 (Session Layer)

* 연결 생성, 유지, 종료

---

### 6계층: 표현 계층 (Presentation Layer)

* 데이터 변환
* 인코딩, 암호화, 압축

---

### 7계층: 응용 계층 (Application Layer)

* 사용자 서비스 (웹, 이메일 등)

---

## 13. 데이터 전송 흐름

### 보내는 과정

1. 응용 계층 (HTTP 요청 생성)

3. 세션 계층 (연결 관리)
