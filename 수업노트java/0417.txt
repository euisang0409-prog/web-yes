Stream API
배열, 컬렉션등에 들어있는 데이터 가공, 필터링 ,변환, 집계하는
등 작업을 더 선언적으로 작성할 수 있게 해주는 기능이다.

※ 선언적
"무엇을 할것인지"를 중심으로 코드를 작성하는 방식
- 어떻게 처리할지(과정)은 신경쓰지 않고 결과로 무엇을 얻고싶은지(목표)만 표현하는 것

List<Integer>list = Arrays.asList(1,2,3,4,5);
List<Interger> result = new ArrayList<>();

기존방식
for(int i = 0; i < list.size(); i++){
	if(list.get(i) % 2 == 0){
	result.add(list.get(i));
	}
}

선언형(결과중심)
- 메서드를 호출해서 결과만 확인하는 방식

왜 Stream API 쓰면 좋은가?
- 코드가 짧아진다.
- 의도가 분명해진다.
-함수형 스타일과 결합되어 유지보수가 쉬워진다.

핵심 특징
1. 원본 데이터를 직접 바꾸지 않는 경우가 ㅁ낳다.
2. 데이터 흐름을 중심으로 작성한다.
3. Stream객체는 일회용이다.
- 한 번 사용한 객체는 다시 사용할 수 없다
4. 지연 연산이 가능하다
- 결과를 바로 주는게 아니라 나중에 돌려주는 작업

Stream API를 사용하는 방법
1. 스트림객체를 생성한다.
- 컬렉션이나 배열을 통해 스트림을 생성
2. 스트림의 데이터를 가공한다.
- 개수를 세거나, 더하거나, 필터링하거나, 변환하거나, 중복을 제거 등등
- 가공은 여러 개를 이어서 사용할 수 있다.
3. 최종연산
- 가공된 결과를 다 더하거나, 개수를 세거나, 출력하거나
가공된 결과를 돌려준다

컬렉션.stream()
	.중간연산()
	.중간연산()
	.중간연산()
	.최종연산();
