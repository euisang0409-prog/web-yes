ㄴ정적내부 클래스
내부 클래스인데 static이 붙은 형태이다.
바깥 객체에 소속된 내부 클래스처럼 보이지만, 실제로는 바깥 객체 없이도 독립적으로 만들수 있는
내부 클래스

중첩클래스
- 인스턴스 내부 클래스 : 외부클래스의 내부에서 만들어지는 클래스
- 정적 내부 클래스 (static) : builder 패턴
- 지역 내부 클래스 : 외부클래스의 메서드 내부에서 만들어지는 클래스
특징 
외부에 노출되지 않아도 된다.

익명 클래스
이름이 없는 클래스
인터페이스와 같이 사용되는 경우가 많다.
인터페이스는 직접 객체를 생성할 수 없기 때문이다.

interface InterA{}

classe Impl implements InterA{...}

class Main{
  mian(){
	Impl impl = new Impl(); 인터페이스를 구현한 클래스의 객체
	InterA inter = new InterA()
		이영역이 익명 클래스 
	}
}

클래스 타입변환
상속
자식타입 -> 부모타입

인터페이스
구현클래스타입 -> 인터페이스타입


- 어떤 메서드를 정의하려고 하는 거지?(먼저 생각할 것)
- 정의 할때는 매개변수에 값이 들어있는게 아님
- 특정 타입의 값만 전달하세요~
- 매개변수에 어떤값이 들어올지 예상하고 안에 기능을 만들어야 된다는 것

- 메서드 구조
접근제한자 반환타입 메서드명(메게변수){
	return 반환값;
}

라이브러리, 모듈
- 클래스, 인터페이스의 모음
- JAR 파일로 존재한다.
- 자주 재사용하는 클래스, 인터페이스, 메서드 같은 것들을 추출해서 관리한다.

모듈
- 패키지 관리 기능까지 포함된다.
- JAR파일을 등록하거나, 프로젝트 자체를 참조하는 방법
