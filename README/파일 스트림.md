# Java File / I/O 스트림 정리

---

# File 클래스
- 파일의 내용이 아니라 파일 자체 정보를 다루는 클래스
- 예: 파일명, 경로, 크기, 존재 여부, 파일/폴더 구분

---

# 경로 개념

## 절대경로
- 윈도우 기준 전체 경로
- 어디서 실행하든 동일한 위치

C:\Users\...\test.txt

---

## 상대경로
- 현재 프로젝트(실행 위치) 기준 경로

./test.txt  
../test.txt

---

# 입출력 (I/O)

## Input (입력)
- 외부 → 프로그램으로 데이터 들어옴

## Output (출력)
- 프로그램 → 외부로 데이터 내보냄

---

# 스트림(Stream)
- 데이터가 흐르는 통로

---

# 1. 바이트 기반 스트림

- 데이터를 1byte 단위로 처리
- 모든 파일 처리 가능 (이미지, 영상, exe, txt 등)

---

## FileInputStream
- 파일 내용을 바이트로 읽는 클래스

### 특징
- 생성자 경로 → FileNotFoundException 발생 가능
- read() → 1바이트 읽기 (IOException 발생 가능)

---

## 문제점
- 한글은 2~3바이트라 깨질 수 있음

---

## 해결 방법

read(byte[] b)  
new String(byte[] b)

- 여러 바이트를 읽고 문자열로 변환

---

## 장점
- 이미지 / 영상 / exe 처리 가능
- 인코딩 영향 없음
- 모든 파일 처리 가능
- 저수준 제어 가능

---

## 단점
- 문자 처리 불편
- 사람이 읽기 어려움

---

# 2. 문자 기반 스트림

- 텍스트 전용 스트림

---

## FileReader
- 파일을 문자 단위로 읽음

FileReader fr = new FileReader("test.txt");  
int data = fr.read();

---

## FileWriter
- 파일에 문자 데이터를 씀

FileWriter fw = new FileWriter("test.txt");  
fw.write("hello");  
fw.close();

---

## 특징
- 자동 문자 처리 (한글 안 깨짐)
- 내부적으로 인코딩 처리됨

---

## 장점
- 한글/텍스트 처리 쉬움
- 코드 간단
- 실무에서 기본 사용

---

## 단점
- 이미지/영상 처리 불가
- 텍스트 전용

---

# 바이트 vs 문자 스트림 비교

| 구분 | 바이트 스트림 | 문자 스트림 |
|------|--------------|-------------|
| 단위 | 1 byte | 1 character |
| 대상 | 모든 파일 | 텍스트 파일 |
| 한글 | 깨질 수 있음 | 정상 처리 |
| 대표 | FileInputStream | FileReader |
| 대표 | FileOutputStream | FileWriter |

---

# 핵심 요약

- File 클래스 → 파일 정보
- Stream → 데이터 흐름
- Byte Stream → 모든 파일 처리
- Character Stream → 텍스트 전용
