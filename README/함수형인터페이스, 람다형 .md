# Java 함수형 인터페이스 정리

---

# 1. Consumer<T>

- 값을 하나 받아서 사용만 함 (출력, 처리)
- 반환값 없음

## 메서드
void accept(T t)

## 예시
Consumer<String> c = str -> System.out.println(str);

---

# 2. Supplier<T>

- 값을 만들어서 반환
- 입력 없음, 출력만 있음

## 메서드
T get()

## 예시
Supplier<Integer> dice = () -> (int)(Math.random() * 6) + 1;

---

# 3. Function<T, R>

- 값을 받아서 다른 값으로 변환

## 메서드
R apply(T t)

## 예시
Function<String, Integer> f = str -> str.length();

---

# 4. Predicate<T>

- 값을 받아서 조건 검사 (true / false)

## 메서드
boolean test(T t)

## 예시
Predicate<String> p = str -> str.length() >= 5;

---

# 5. BiConsumer<T, U>

- 값을 두 개 받아서 사용 (출력 등)
- 반환값 없음

## 메서드
void accept(T t, U u)

## 예시
(name, age) -> System.out.println(name + age);

---

# 6. BiPredicate<T, U>

- 값을 두 개 받아서 비교 후 true/false 반환

## 메서드
boolean test(T t, U u)

## 예시
(a, b) -> a.equals(b);

---

# 핵심 차이 정리

| 인터페이스 | 입력 | 출력 |
|-----------|------|------|
| Consumer  | 1개  | 없음 |
| Supplier  | 없음 | 1개 |
| Function  | 1개  | 1개 |
| Predicate | 1개  | boolean |
| BiConsumer| 2개  | 없음 |
| BiPredicate| 2개 | boolean |

---

# 핵심 포인트

- Consumer → 출력/처리
- Supplier → 생성
- Function → 변환
- Predicate → 조건 검사
