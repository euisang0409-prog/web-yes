# Java 람다식(Lambda Expression) 정리

---

# 1. 기존 방식 vs 함수형 프로그래밍

## 기존 방식 (객체지향)
1. 클래스 생성
2. 클래스 안에 메서드 작성
3. 객체 생성
4. 객체로 메서드 호출

---

## 함수형 프로그래밍
- 함수 자체를 값처럼 사용
- 객체 생성 없이 바로 호출 가능
- 다른 파일에 정의된 함수도 import 후 사용 가능

---

# 2. 람다식 장점

1. 코드가 간결해짐
2. 핵심 로직만 작성 가능

---

# 3. 함수형 인터페이스

람다식을 사용하려면 반드시 필요

조건:
- 추상 메서드가 1개만 존재해야 함

---

# 4. 람다식 기본 문법

(매개변수) -> { 실행문 }

특징:
- 반환 타입 생략
- 메서드 이름 없음

---

# 5. 매개변수 규칙

## 1개일 때
- 소괄호 생략 가능
- 타입 생략 가능

예:
x -> x * 2

---

## 2개 이상일 때
- 소괄호 생략 불가능
- 타입 생략하려면 전부 생략해야 함

예:
(a, b) -> a + b

잘못된 예:
(int a, b) -> a + b   // 에러

---

# 6. 실행문 규칙

## 1줄일 때
- 중괄호 생략 가능

예:
(a, b) -> a + b

---

## return 포함 1줄일 때
- return + 중괄호 같이 생략 가능

예:
(a, b) -> a + b

---

## 여러 줄일 때
- 중괄호 필요
- return 필요

예:
(a, b) -> {
    int result = a + b;
    return result;
}

---

# 7. 람다식 활용

## 1. 변수에 담기

Calculator cal = (a, b) -> a + b;

---

## 2. 변수로 호출

cal.calc(3, 2);

---

## 3. 매개변수로 전달

public void execute(Calculator c) {
    c.calc(1, 2);
}

---

## 4. 반환값으로 사용

public Calculator getCalc() {
    return (a, b) -> a + b;
}

---

# 8. 핵심 요약

람다식 = 함수를 간단하게 표현한 것

- 객체 없이 함수 사용 가능
- 함수형 인터페이스 필요
- 매개변수 규칙 존재
- 실행문 1줄이면 생략 가능
- 함수 자체를 변수, 매개변수, 반환값으로 사용 가능
