1 개념
자바에서 String 객체는 한 번 생성되면 그 값을 변경할 수 없다.
이를 문자열의 불변성(Immutable)이라고 한다.

2 예제 코드 
String str = "Hello";
str =str + " world";

3 동작원리
위 코드는 기존 문자열이 변경되는 것이 아니라,
"Hello" 문자열 생성
" Hello World" 라는 새로운 문자열 생성
변수 str 이 새로운 문자열을 가리킴
 기존 " Hello" 는 그대로 남아 있음

4 메모리 구조 설명
String은 힙(Heep) 영역의 문자열 상수 풀(String Pool)에 저장됨
같은 문자열은 공유하지만,
내용이 바뀌면 무조건 새로운 객체 생성

5. 문제점
String s = "";
for (int i = 0; i < 10000 i++) {
	s  += i;
}
-> 반복할 때마다 새로운 문자열 생성
-> 메모리 낭비 + 성능 저하 발생

6. 해결 방법
StringBuilder sb = new StringBuilder();

for ( int i = 0; i < 10000; i++) {
	sb.append(i);
}
String result = sb.toString();

-> StringBuilder는 가변(mutable) 객체
-> 문자열을 변경해도 새 객체를 만들지 않음

정리
String은 불변 객체 (Immutable)
값 변경 시 새로운 객체 생성
반복적인 문자열 연산 시 비효율적
해결 StringBuilder 사용

요약
자바의 String은 불변 객체이므로 값이 변경될 때 기존 객체를 수정하지 않고 새로운 객체를 생성한다.
