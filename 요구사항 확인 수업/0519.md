# 3. 요구사항 확인

- 분석가가 작성한 요구사항이 올바른지 검증하는 과정
- 잘못 해석하면 시간 낭비와 비용 증가로 이어질 수 있다.

---

# 확인과 검증

## 확인
- 고객의 목적과 요구사항이 맞는지 확인

## 검증
- 문서가 올바르게 작성되었는지 검사하는 것

---

# 검증 시 확인 항목

## 1. 이해 가능성
- 읽는 사람이 같은 의미로 이해할 수 있는가?
- 애매한 표현은 사람마다 다르게 해석될 수 있다.

### 잘못된 예
사용자는 "빠르게" 로그인 할 수 있어야 한다.

### 수정 예
사용자는 로그인 버튼 클릭 후 3초 이내에 로그인 결과를 확인할 수 있어야 한다.

---

## 2. 일관성
- 서로 모순된 요구사항은 없는가?

### 요구사항 1
회원은 로그인 없이 게시글을 작성할 수 있다.

### 요구사항 2
로그인한 회원만 게시글 작성 가능

### 문제점
- 두 요구사항이 서로 충돌함
- 하나만 선택해야 한다.

---

## 3. 완전성
- 필요한 정보가 빠져있지 않은가 확인

### 요구사항
사용자는 회원가입 할 수 있어야 한다.

### 누락된 정보
- 입력 항목
- 비밀번호 조건
- 중복 확인 여부
- 실패 처리 방식

---

## 4. 표준 준수
- 회사나 프로젝트의 문서 작성 규칙을 따르는가

### 문제점
표준이 없으면 사람마다 형식이 달라져 이해하기 어렵다.

# 요구사항 문제점 예시

## 문제
사용자는 편리하게 로그인 할 수 있다.

## 문제점
- "편리하게"가 모호함
- 측정 불가능
- 테스트 어려움

## 수정
- 사용자는 아이디와 비밀번호를 입력하여 로그인할 수 있어야 한다.
- 로그인 실패 시 오류 메시지를 출력해야 한다.
