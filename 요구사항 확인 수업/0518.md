# 운영체제 선정 시 고려사항

## 1. 신뢰도
- 장기간 운영 시 안정성
- 메모리 누수 여부
- 보안 패치 재기동 빈도
- 운영체제 버그 여부

## 2. 성능
- 대규모 사용자 처리 가능 여부
- 대용량 파일 처리
- 지원 메모리 크기(32bit/64bit)

## 3. 기술지원
- 공급 벤더 기술지원
- 사용자 정보 공유
- 오픈소스 여부

## 4. 주변기기
- 설치 가능한 하드웨어
- 프린터, USB, 그래픽카드 지원 여부

## 5. 구축비용
- 하드웨어 비용
- 라이선스 비용
- 유지보수 비용
- 서버 전기세
- AWS 사용료
- 관리자 인건비
- 데이터 백업 비용

## 6. 총 소유 비용(TCO)
- 시스템 전체 운영 기간 동안 발생하는 총 비용

# 라이선스

## GNU AGPL v3.0
- GPL보다 강력
- 서버 실행 서비스도 소스 공개 의무 가능

## BSD
- 상업적 사용 가능
- 수정 및 판매 가능
- 소스 공개 의무 없음
- 저작권 표시 유지 필요

---

# DBMS 고려사항

## 가용성
- 장애 발생 가능성

- 백업 및 복구

## 성능
- 대규모 데이터 처리

- 튜닝 옵션 지원

- 벤더 기술지원
- 사용자 커뮤니티
- 오픈소스 여부

## 상호 호환성
- 지원 운영체제
- JDBC 지원 여부

## 구축비용
- 라이선스 비용
- 유지관리 비용
- TCO

# Validation vs Verification

| 구분 | Validation | Verification |
|---|---|---|
| 의미 | 고객 요구사항 확인 | 문서 품질 검증 |

# Validation vs Verification 차이

| Validation | Verification |
|---|---|
| 고객 요구사항 확인 | 문서 품질 검증 |
| 고객이 원하는가 | 문서가 올바른가 |
| 요구사항 적합성 | 요구사항 정확성 |
