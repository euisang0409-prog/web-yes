요구사항 확인

1. 현행 시스템의 분석
- 새 시스템을 만들거나 기존 시스템을 개선하기 위해 현재 사용중인 업무 방식, 프로그램, 데이터, 서버, 네트워크, 문제점을 조사하고 정리하는 활동

목적
- 새로운 시스템을 정확하게 만들기 위해 현재 상황을 이해한다.


문제점
1. 기존 업무의 흐름을 모른다면 필요한 기능을 빠트릴 수 있다.
2. 기존 데이터 구조를 모른다면 데이터 이전 시 오류 발생
3. 기존 시스템 연동을 모른다면 외부 시스템과 연결 실패
4. 사용자 불편 사항을 모른다면 새 시스템을 만들어도 만족도가 낮다.
5. 현재 원인을 모른다면 문제 해결이 아닌 단순 재개발이 된다.

문제 1. 운영체제의 역할 설명하기

운영체제(OS)의 역할을 설명하고, 운영체제가 없는 경우 어떤 문제가 발생하는지 작성하시오.

운영체제 역할
- 하드웨어, 소프트웨어 자원을 관리한다.
- CPU, 메모리, 디스크 등을 프로그램에 할당한다.
- 사용자와 컴퓨터 사이의 중간관리자 역할을 수행한다.

운영체제가 없는 경우 문제
- 프로그램 실행이 어렵다.
- CPU와 메모리를 직접 제어해야 한다.
- 여러 프로그램을 동시에 실행하기 어렵다.


문제 2. 운영체제 종류 구분하기

Windows
- 개인용, 중소규모 서버에 많이 사용된다.
- GUI를 사용한다.

Linux
- 오픈소스이다.
- 서버 환경에서 많이 사용된다.

UNIX
- 안정성이 높고 대규모 서버 환경에 사용한다.


문제 3. 신뢰도 요구사항 분석하기

- 서버가 장기간 안정적으로 동작해야하기 때문이다.
- 운영체제 장애 발생 가능성을 줄여야 한다.
- 메모리 누수 발생 시 성능 저하가 발생할 수 있다.
- 보안 패치 및 버그 수정 시 안정적인 운영이 필요하다.
- 서버 중단 시 결제 오류와 서비스 장애가 발생할 수 있다.


문제 4. 메모리 누수 설명하기

개념
- 프로그램이 사용한 메모리를 반납하지 않는 현상

문제점
- 메모리 사용량이 계속 증가한다.
- 서버 속도가 느려진다.
- 프로그램 오류가 발생할 수 있다.
- 결국 서버를 재기동할 필요가 있다.


문제 5. 동시 사용자 처리 분석하기

- CPU 사용량 분배
- 메모리 할당
- 네트워크 연결 관리
- 프로세스 스케줄링
- 사용자 요청 처리 순서 결정


문제 6. 대용량 파일 처리 요구사항 분석하기

- 서버 과부하가 발생할 수 있다.
