# 요구사항 확인

## 1. 현행 시스템의 분석

- 새 시스템을 만들거나 기존 시스템을 개선하기 위해 현재 사용 중인 업무 방식, 프로그램, 데이터, 서버, 네트워크, 문제점을 조사하고 정리하는 활동

### 목적
- 새로운 시스템을 정확하게 만들기 위해 현재 상황을 이해한다.

---

## 문제점

1. 기존 업무 흐름을 모르면 필요한 기능을 빠트릴 수 있다.
2. 기존 데이터 구조를 모르면 데이터 이전 시 오류가 발생한다.
3. 기존 시스템 연동을 모르면 외부 시스템 연결에 실패할 수 있다.
4. 사용자 불편 사항을 모르면 새 시스템 만족도가 낮아질 수 있다.
5. 현재 문제 원인을 모르면 단순 재개발이 될 수 있다.

---

# 현행 시스템 파악 절차

## 1단계 : 현행 시스템의 구성, 기능, 인터페이스 현황 파악

- 현재 시스템이 무엇으로 구성되어 있고 어떤 기능을 제공하며 다른 시스템과 어떻게 연결되어 있는지 파악

---

## 시스템 구성 현황 파악

- 현재 운영 중인 시스템의 종류와 역할을 정리
- 각 시스템이 어떤 업무를 담당하는지까지 정리

### 학원 수강관리 시스템

| 시스템명 | 주요 사용자 | 역할 |
|---|---|---|
| 수강생 관리 시스템 | 관리자 | 수강생 등록, 수정, 조회 |
| 출결관리 시스템 | 강사, 관리자 | 출결 관리 |
| 상담관리 시스템 | 관리자(상담직원) | 상담 이력 기록 |
| 결제관리 시스템 | 관리자 | 수강료 결제 내역 관리 |
| 문자 발송 시스템 | 관리자 | 공지 문자 발송 |

---

## 기능 현황 파악

- 각 시스템이 제공하는 기능을 정리

### 수강생 관리 시스템 기능

- 수강생 등록 : 신규 수강생 등록
- 수강생 조회 : 수강생 정보 조회
- 수강생 수정 : 수강생 정보 수정
- 수강생 삭제 : 잘못 등록된 정보 삭제

---

## 인터페이스 현황 파악

- 시스템과 시스템이 데이터를 주고받는 연결 지점

### 쇼핑몰 예시

- 결제 API → 주문번호, 결제금액, 결제상태
- 택배사 API → 주문번호, 주소, 송장번호
- 문자 발송 API → 전화번호, 메시지 내용
- 이메일 API → 이메일 주소, 인증 코드

---

# 문제 1. 운영체제(OS)의 역할

## 역할
- 하드웨어 및 소프트웨어 자원 관리
- CPU, 메모리, 디스크 자원 할당
- 사용자와 컴퓨터 사이 중간 관리자 역할

## 운영체제가 없는 경우 문제점
- 프로그램 실행 어려움
- CPU와 메모리를 직접 제어해야 함
- 여러 프로그램 동시 실행 어려움

---

# 문제 2. 운영체제 종류 특징

## Windows
- 개인용 및 중소규모 서버 사용
- GUI 환경 제공

## Linux
- 오픈소스
- 서버 환경에서 많이 사용

## UNIX
- 안정성이 높음
- 대규모 서버 환경 사용

---

# 문제 3. 신뢰도 요구사항 분석

## 운영체제 신뢰도가 중요한 이유
- 서버가 장기간 안정적으로 동작해야 함
- 장애 가능성 감소 필요
- 메모리 누수 발생 시 성능 저하 가능
- 보안 패치 및 버그 수정 안정성 필요
- 서버 중단 시 결제 오류 및 서비스 장애 발생 가능

---
