# 1. 현행 시스템 분석

## 개념
새 시스템을 만들거나 기존 시스템을 개선하기 전, 현재 사용 중인 업무 방식, 프로그램, 데이터, 서버, 네트워크, 문제점을 조사하고 정리하는 활동

## 분석을 하지 않았을 때 문제점
- 필요한 기능 누락
- 데이터 이전 오류 발생
- 외부 시스템 연동 실패
- 사용자 만족도 저하
- 단순 재개발 가능성 증가

---

## 절차

### 1단계 : 현행 시스템 구성, 기능, 인터페이스 현황 파악
- 현재 시스템 구성 확인
- 제공 기능 확인
- 외부 시스템 연결 상태 확인

### 2단계 : 현행 시스템 아키텍처 및 소프트웨어 구성 현황 파악
- 시스템 구조 확인
- 사용 소프트웨어 확인

### 3단계 : 하드웨어 및 네트워크 분석
- 물리 서버 또는 클라우드 환경 분석
- 온프레미스 / 클라우드 환경 확인
