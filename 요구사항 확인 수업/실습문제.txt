# 3. 클라우드 vs 온프레미스

| 구분    | 클라우드  | 온프레미스 |
| ----- | ----- | ----- |
| 구축 위치 | 외부    | 내부    |

| 유지보수  | 업체 담당 | 직접 관리 |
| 확장성   | 쉬움    | 어려움   |

## AWS 대표 서비스

* EC2 : 가상 컴퓨터
* S3 : 저장소

# 4. 요구사항 유형

| 유형       | 설명                    |
| -------- | --------------------- |
| 기능 요구사항  | 시스템이 수행해야 하는 기능       |
| 비기능 요구사항 | 성능 / 보안 / 안정성 등 품질 요구 |
| 제약사항     | 반드시 지켜야 하는 조건         |

## 예시

* 회원은 로그인 할 수 있어야 한다. → 기능
* 검색 결과는 2초 이내 출력되어야 한다. → 비기능(성능)
* 비밀번호는 암호화하여 저장해야 한다. → 비기능(보안)
* 데이터베이스는 MySQL을 사용해야 한다. → 제약사항

# 8. 요구사항 문제점 분석

## 문제

사용자는 편리하게 로그인 할 수 있다.

## 문제점

* 편리하게의 기준이 모호함
* 테스트 불가능
* 측정 어려움

## 수정 예시

* 사용자는 아이디와 비밀번호를 입력하여 로그인할 수 있어야 한다.
* 로그인 실패 시 오류 메시지를 출력해야 한다.

# 31. 자주 나오는 서술형

## 기능 요구사항과 비기능 요구사항 차이

기능 요구사항은 시스템이 수행해야 하는 기능을 의미한다.
비기능 요구사항은 성능, 보안, 안정성 등의 품질 속성을 의미한다.

## 클라우드와 온프레미스 차이

클라우드는 외부 서버 기반 방식이며 확장성이 높고 초기 비용이 낮다.
온프레미스는 자체 서버 구축 방식이며 보안 통제가 쉽지만 구축 비용이 높다.

## ESB 특징

서비스 중심 구조이며 느슨한 결합을 지원한다.
버스 구조 기반으로 확장성이 높다.

  → 완전성 / 일관성 / 명확성 / 추적성 / 검증 가능성

* 클라우드

* EAI
  → 중앙 집중형

* ESB
  → 느슨한 결합 / 버스 구조

* 미들웨어
  → OS와 응용프로그램 사이 소프트웨어
