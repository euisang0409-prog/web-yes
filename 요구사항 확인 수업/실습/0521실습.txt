# 온라인 도서 예약 시스템 요구사항 확인

A교육기관은 기존 전화 예약 방식 대신 온라인 도서 예약 시스템을 개발하려고 한다.
분석가는 사용자 면담 후 다음과 같은 요구사항 목록과 유스케이스 명세서를 작성했다.

---

# 1. 요구사항 목록

| ID      | 이름       | 유형  | 우선순위 | 중요도 | 내용                             | 수용 여부 |
| ------- | -------- | --- | ---- | --- | ------------------------------ | ----- |
| REQ-001 | 회원 로그인   | 기능  | 상    | 10  | 회원은 아이디와 비밀번호를 이용하여 로그인할 수 있다. | 수락    |
| REQ-002 | 도서 검색    | 기능  | 상    | 10  | 회원은 제목 또는 저자로 도서를 검색할 수 있다.    | 수락    |
| REQ-003 | 도서 예약    | 기능  | 상    | 9   | 회원은 대출 가능한 도서를 예약할 수 있다.       | 수락    |
| REQ-004 | 예약 알림 발송 | 기능  | 중    | 7   | 예약 완료 시 문자 메시지를 발송한다.          | 수락    |
| REQ-005 | 시스템 응답속도 | 비기능 | -    | -   | 시스템은 빠르게 동작해야 한다.              | 수락    |
| REQ-006 | 서버 가용성   | 비기능 | -    | -   | 서비스는 안정적으로 운영되어야 한다.           | 수락    |

---

# 2. 유스케이스 명세서 작성 현황

유스케이스명

* 회원 로그인
* 도서 검색

※ 도서 예약(REQ-003), 예약 알림 발송(REQ-004) 명세서는 아직 작성되지 않았다.

---

# 3. 인터페이스 요구사항

예약 알림 기능은 외부 문자 시스템과 연동된다.

| 항목       | 내용            |
| -------- | ------------- |
| 시스템명     | SMS 발송 시스템    |
| 제공자      | ABC 문자 서비스    |
| 요청자      | 온라인 도서 예약 시스템 |
| 연계 방법    | REST API      |
| 인터페이스 구분 | 외부            |
| 호출 주기    | 예약 완료 즉시      |
| 계약 체결 여부 | 미체결           |
| 제약사항     | 없음            |

---

# 4. 사용자 면담 기록

* 사용자는 예약 후 문자 수신까지 5초 이상 걸리면 불편하다고 응답하였다.
* 관리자는 장애 발생 시에도 서비스를 계속 사용할 수 있기를 원한다.

단, 도서 예약 프로세스는 다음 순서를 반드시 따른다.

로그인
→ 도서 검색
→ 도서 예약

→ 예약 완료 알림
