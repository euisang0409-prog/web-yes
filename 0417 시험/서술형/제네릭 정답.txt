제네릭 (Generic)

1. 제네릭의 개념과 사용 목적
   제네릭(Generic)은 클래스나 메서드에서 사용할 데이터 타입을 미리 지정하지 않고, 객체를 생성하거나 메서드를 호출할 때 타입을 결정하는 기능이다. 이를 통해 하나의 코드로 다양한 타입을 처리할 수 있으며 코드의 재사용성을 높일 수 있다.

2. 타입 안정성과의 관계
   제네릭을 사용하면 컴파일 시점에 타입 검사가 이루어져 잘못된 타입 사용을 미리 방지할 수 있다. 따라서 실행 시 발생할 수 있는 오류를 줄일 수 있어 타입 안정성이 향상된다.

3. 제네릭 클래스와 제네릭 메서드의 차이
   제네릭 클래스는 클래스 전체에 타입을 적용하는 것으로, 객체 생성 시 타입이 결정된다. 반면 제네릭 메서드는 특정 메서드에서만 타입을 적용하며, 메서드를 호출할 때 타입이 결정된다.

4. 사용하지 않았을 때 문제점
   제네릭을 사용하지 않으면 Object 타입을 사용해야 하므로 형변환이 필요하다. 이 과정에서 잘못된 형변환으로 인해 런타임 오류(ClassCastException)가 발생할 수 있으며, 코드의 가독성과 안정성이 떨어진다.
