컬렉션 프레임워크 (Collection Framework)

1. List, Set, Map의 특징
   List는 순서가 있고 중복을 허용한다.
   Set은 순서가 없고 중복을 허용하지 않는다.
   Map은 Key와 Value의 쌍으로 데이터를 저장하며 Key는 중복을 허용하지 않고 Value는 중복을 허용한다.

2. 대표 구현 클래스
   List → ArrayList, LinkedList
   Set → HashSet
   Map → HashMap

3. 언제 사용하면 유리한지
   List는 데이터의 순서가 중요하거나 중복 데이터를 저장할 때 유리하다.
   Set은 중복을 제거해야 할 때 사용하기 적합하다.
   Map은 키를 이용해 데이터를 빠르게 검색하고 관리할 때 유리하다.
