# 정규화 vs 반정규화 한눈에 비교

| 구분 | 정규화 (Normalization) | 반정규화 (Denormalization) |
|---|---|---|
| 핵심 목적 | 데이터의 무결성 및 일관성 유지 | 데이터 조회 성능 향상 |
| 설계 방향 | 테이블을 세분화 (나눔) | 테이블을 통합/분할/추가 (합침, 복제) |
| 작동 원리 | 중복을 제거하여 데이터 모델 최적화 | 성능을 위해 의도적으로 중복 허용 |
| 장점 | 데이터 이상(Anomaly) 방지, 유지보수 용이 | 조회(SELECT) 속도 향상, JOIN 감소 |
| 단점 | 테이블 증가로 JOIN 연산 복잡 가능 | 데이터 일관성 유지 어려움, 갱신 비용 증가 |

---

# 핵심 요약

## 정규화
- 중복 제거 중심
- 데이터 무결성 유지
- INSERT / UPDATE / DELETE 이상 방지
- JOIN 증가 가능

## 반정규화
- 조회 성능 중심
- JOIN 감소 목적
- 중복 데이터 허용
- UPDATE 시 관리 비용 증가

# 시험 핵심 한줄

- 정규화 : "데이터 중복 제거와 무결성 유지 목적"
- 반정규화 : "조회 성능 향상을 위해 중복 허용"
