# 물리 데이터 저장소 설계 정리

## 1. 관계형 데이터베이스 제약조건

### PK - FK 제약조건

PK(Primary Key)와 FK(Foreign Key)로 연결된 두 테이블 간의 데이터 무결성을 유지하기 위한 제약조건이다.

### ON DELETE

#### CASCADE

PK 데이터가 삭제되면 FK 데이터도 함께 삭제된다.

#### RESTRICT

FK에서 사용 중인 경우 PK 데이터를 삭제할 수 없다.

#### NULLIFY

PK 데이터가 삭제되면 FK 컬럼 값을 NULL로 변경한다.

### ON UPDATE

#### CASCADE

PK 값이 수정되면 FK 값도 함께 수정된다.

#### RESTRICT

FK에서 사용 중인 경우 PK 값을 수정할 수 없다.

#### NULLIFY

PK 값이 수정되면 FK 값을 NULL로 변경한다.

### 사용 시점

외래키(FK) 제약조건 생성 시 함께 설정한다.

# 2. 인덱스(Index) 설계

조회 성능 향상을 위해 특정 컬럼에 생성하는 객체이다.

## 인덱스 선정 기준

### 분포도

일반적으로 10~15% 이내가 적절하다.

분포도 계산식

분포도 = (1 / 도메인 수) × 100

분포도가 낮을수록 인덱스 효율이 높다.

### WHERE 절에서 자주 사용하는 컬럼

조회 조건으로 자주 사용되는 컬럼에 적용한다.

### 자동 생성되는 경우

PK와 UNIQUE 제약조건을 지정하면 인덱스가 자동 생성된다.

### 수정이 적은 컬럼

INSERT, UPDATE가 빈번한 컬럼은 인덱스 효율이 떨어질 수 있다.

### 결합 인덱스

자주 함께 사용되는 컬럼은 복합 인덱스로 생성한다.

# 3. 뷰(View)

SELECT 결과를 저장한 가상의 테이블이다.

## REPLACE

기존 뷰가 존재하면 새 정의로 다시 생성한다.

```sql
CREATE OR REPLACE VIEW 뷰이름 AS
SELECT ...
```

## FORCE

기본 테이블이 없어도 뷰 생성 가능

Oracle에서 사용

실행 시 기본 테이블이 없으면 오류 발생

## NOFORCE

기본 테이블이 존재할 때만 뷰 생성 가능

## WITH CHECK OPTION

뷰의 조건을 만족하는 데이터만 INSERT 또는 UPDATE 가능

## WITH READ ONLY

읽기 전용 뷰 생성

## 뷰 설계 시 고려사항

* 데이터가 많으면 조회 성능 저하 가능
* 조건 컬럼에 인덱스 존재 여부 확인

# 4. 클러스터(Cluster)

관련 데이터를 물리적으로 가까운 위치에 저장하여 조회 성능을 향상시키는 기법

## 특징

* 자주 함께 조회되는 데이터에 적용
* 분포도가 넓은 경우 유용
* 여러 블록에 저장되는 대용량 데이터에 적합
* 디스크 이동 감소
* 조회 성능 향상

## 단점

* INSERT
* UPDATE
* DELETE

성능이 저하될 수 있다.

## 적용 고려 대상

다음 연산이 빈번한 컬럼

* UNION
* DISTINCT
* ORDER BY
* GROUP BY

# 5. 파티션(Partition) 설계

하나의 테이블을 여러 저장 영역으로 분할하여 관리하는 기술

## 범위 분할(Range Partition)

사용자가 기준을 직접 설정

예)

* 2024년 데이터
* 2025년 데이터

## 해시 분할(Hash Partition)

DBMS가 자동으로 데이터를 균등 분배

## 조합 분할(Composite Partition)

파티션 내부에 서브 파티션 생성

# 8. MySQL vs Oracle 자동 증가

## MySQL

```sql
CREATE TABLE MEMBER(
    member_id INT AUTO_INCREMENT
);
```

## Oracle

```sql
CREATE SEQUENCE member_seq
START WITH 1
INCREMENT BY 1;
```

```sql
INSERT INTO MEMBER
VALUES(member_seq.NEXTVAL);
```

# 9. 트랜잭션(Transaction)

여러 작업을 하나의 작업 단위로 처리하는 기능

예)

1. 내 계좌 잔고 차감
2. 상대방 계좌 잔고 증가

## COMMIT

변경 내용을 실제 데이터베이스에 반영

```sql

```

## ROLLBACK

작업을 이전 상태로 복구

```sql
ROLLBACK;
```
